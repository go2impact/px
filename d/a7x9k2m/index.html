How It Works

The system is redundant by design. Two independent collectors watch the same channels. As long as one of them sees the message, it makes it into the database.

1. Intake (redundant)

A dedicated Observer WhatsApp account sits in all 70 channels. On the Mac running the monitoring stack, two collectors watch it simultaneously:

A
Native notifications macOS desktop notifications from WhatsApp. Fast wake-up signal. Can truncate long messages.
B
Baileys (linked device) Direct WhatsApp protocol connection. Richer data — full message text, sender identity, group metadata. Stronger evidence.

Both feed into the same local SQLite database. The merge layer reconciles them — if one catches something the other missed, it backfills. If both catch it, it confirms.

2. Classify

Every message gets checked against your three rules. The AI reads the message, evaluates it, and if there's a violation, drafts a custom moderation response appropriate to the tone and context of the conversation.

3. Route & Alert

This version is set up on a Mac, so it uses iMessage to send the alert. When a violation is flagged, the system looks at who's supposed to be on coverage at that time — based on schedule, time of day, whatever routing rules you set — and sends them a text with the flagged message plus the suggested response.

If the person doesn't acknowledge it within a set window, it can re-send or escalate to a second person. The routing logic is configurable.

It never posts back into WhatsApp automatically. That's what breaks. The system watches, classifies, drafts, and alerts. A human reviews and posts the response themselves. The only time anyone should touch the Observer account directly is once a week or so — just open WhatsApp on the phone briefly to keep the session alive. It won't log out often, but it's good hygiene.

Troubleshooting Checklist

The phone is the main account. The Mac only helps surface activity. If the phone is offline or logged out, monitoring can stop.

Common causes when alerts stop

  • Phone offline or out of power
  • WhatsApp asking to relink
  • Mac in Focus Mode or Do Not Disturb
  • Observer chat left open on the Mac (suppresses notifications)
  • Observer account being used like a normal working account

Check in this order

  1. Check the phone first. Make sure it is powered, connected, and still logged into WhatsApp.
  2. Check the Mac. Make sure Focus / Do Not Disturb is off.
  3. Make sure WhatsApp Desktop is not sitting open on the observer chat.
  4. If needed, open WhatsApp on the phone, go to Settings > Linked Devices, and relink the account to the Mac. Then test again.

Crucial Mac rule: Never leave the observer chat open and active in WhatsApp Desktop. Close it or move away from it so it is not the active conversation. If that chat is open in the foreground, the Mac may not trigger the notifications the system needs.

Escalate if

  • The phone keeps getting forced to relink
  • The phone has power and internet but alerts still don't appear
  • The account stops holding a connection
  • Someone needs to change the actual setup, not just relink the phone
How the Rulebook Should Think

The collectors are not the moderation brain. This is the policy layer that tells the system what counts, what is ambiguous, and what should happen next.

Default moderation logic

  1. Decide whether the event is a real member message or system noise.
  2. Check whether the message touches one or more rule categories.
  3. Measure the evidence quality: notification-only, provisional, or confirmed.
  4. Choose the route: ignore, log, review now, human warning, or admin escalation.
  5. If a private follow-up is appropriate, decide whether that should happen off-platform by text.

Rule buckets

1
Real-name policy Anonymous, troll-like, or clearly fake identities. Start gentle -- this is usually a reminder before it becomes enforcement.
2
Solicitation / promotion Pitches, product pushes, deal promotion, direct commercial asks. Keep generic until real gray areas surface.
3
Respect / hostility Personal attacks, escalation spirals, abusive tone. Separate normal disagreement from direct hostility.

Routing outcomes

  • ignore -- nothing meaningful happened
  • log_only -- record it, don't distract an operator
  • review_pending -- suspicious enough to watch, not strong enough to act
  • review_now -- strong enough that a human should look immediately
  • human_warning -- operator should send a reminder or warning
  • off_platform_message -- text follow-up may be easier than WhatsApp DM
  • escalate_admin -- involve an admin immediately

Why keep it generic first: The temptation is to overfit to exactly three rules. Resist that. The starter layer should make room for more categories, clearer thresholds, and operator feedback without a rebuild.

How the Database is Meant to Grow

The starter kit keeps the database practical. Raw evidence stays factual, packet rows hold the operational interpretation, and the schema is shaped so a dashboard can sit on top without rewriting the stack.

Three layers, three jobs

1
Source events Raw intake from notifications and the richer collector. This is evidence, not judgment.
2
Canonical events Merged message records with provenance. Still factual. Should not become the policy brain.
3
Operator packets The action layer. Rule signals, violation guesses, routing, and dashboard-friendly columns live here.

What belongs on packet rows

  • thread_key, group_name, sender_key -- so events can be routed and joined
  • primary_rule_id, violation_guess -- so dashboards can filter by moderation issue
  • overall_assessment, source_confidence -- to separate strong cases from wake-up signals
  • human_review_status, human_final_disposition -- the bridge into human workflow
  • external_person_key -- for future CRM joins and repeat-offender tracking

Repeat offenders and member joins

Repeat-offender views group by sender_key when it exists. CRM joins eventually map the WhatsApp identity to your member record. The schema is already pointed toward this -- start with packet-level rollups, enrich later.
